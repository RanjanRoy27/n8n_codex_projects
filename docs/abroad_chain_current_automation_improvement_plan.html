3. Current Automation Summary

Workflow Facts

  • 22-node n8n workflow.
  • Workflow ID: 0faiT7g7k8PI3kVI.
  • Local URL: http://localhost:5678/workflow/0faiT7g7k8PI3kVI.
  • Inactive at review time.
  • Timezone: Asia/Dhaka.
  • Schedule: 9:00 AM, Monday to Saturday.

Current Flow

  • Manual or scheduled trigger starts the run.
  • Brand Config provides central settings.
  • Google Sheets supplies performance and calendar context.
  • HTTP/RSS nodes collect public research signals.
  • OpenAI nodes generate topic, caption, visual brief, and QA decision.
  • IF node routes approved vs revision-needed content.
  • Google Sheets stores the final pack, lead-log task, and tomorrow plan.
  • Set node creates the human Meta Business Suite handoff.

4. Validation Findings

Status: the exported workflow and live workflow validation both showed valid structure with 0 blocking errors.

Main Warnings To Address

  • AI APIs can hit rate limits or return errors.
  • Google Sheets resource locators are missing useful cachedResultName metadata.
  • Google Sheets append nodes should review value formatting behavior.
  • The approval IF branch should use stronger structured parsing and clearer error routing.
  • The workflow is a long linear chain and may later benefit from sub-workflows.

These are production-readiness warnings, not blockers. The immediate opportunity is hardening and output quality, not rebuilding from scratch.

5. Immediate Improvement Scope

  1. Add complete prompts to all four OpenAI nodes.
  2. Improve structured AI output handling, ideally with a parsing/normalization step before downstream logic.
  3. Strengthen the approval gate so it checks a parsed boolean rather than searching raw text.
  4. Improve Google Sheets reliability and confirm exact spreadsheet/tab/column mappings.
  5. Improve the final content pack structure so humans can use it without digging through raw AI output.
  6. Keep Meta/Canva as a human handoff in this phase.

6. SOP Insights To Use Now

These SOP rules are valuable enough to apply immediately without expanding into the full SOP automation.

  • Bangla is the primary caption language.
  • Every post must have exactly one CTA.
  • No false, unverifiable, or guaranteed claims.
  • No competitor comparison or competitor naming in content.
  • Human approval is required before publishing.
  • Audience should feel local to Rangpur and Lalmonirhat.
  • Brand style should use Navy and Gold, with Teal only as an accent.
  • Use simplified content pillars for now, then expand toward the SOP's full 10-pillar system later.

7. Later Upgrade Scope

These are intentionally later-phase items because they require additional API access, compliance setup, credentials, or operational process maturity.
  • Meta API insights for reach, engagement, comments, and post performance.
  • Meta publishing or scheduling automation.
  • Canva API or design automation.
  • Notion, Airtable, or CRM migration.
  • Automated comment and DM lead capture.
  • Weekly KPI report automation.
  • Full 10-pillar daily content calendar and self-improvement loop.

8. Serial Implementation Plan

  1. Finalize client inputs and Google Sheet. Confirm the real Google Sheet ID, tab names, column names, CTA wording, services, locations, target countries, and banned claims.
  2. Patch Brand Config. Replace placeholders and add practical brand fields the prompts need, while keeping credentials inside n8n.
  3. Add structured prompts to AI nodes. Give each OpenAI node a clear role, input context, SOP rules, and required JSON output shape.
  4. Add or strengthen output parsing. Normalize AI responses into stable fields such as topic, pillar, caption, visual_brief, approved, and fixes_needed.
  5. Fix approval logic. Route based on parsed approved === true, not string matching raw model text.
  6. Improve Sheets append fields. Store separate structured columns while keeping a readable Final Pack field for the human scheduler.
  7. Validate exported JSON. Use n8n workflow validation with runtime profile before touching the live workflow.
  8. Import or update n8n. After review, update the live workflow 0faiT7g7k8PI3kVI or import a revised copy.
  9. Manual test with sample data. Run from Manual Test Trigger using a sample Performance Log row and valid credentials.
  10. Activate only after successful test. Confirm rows append correctly, QA routes correctly, and human handoff is clear.

9. Testing Checklist

  • Manual trigger succeeds.
  • Research pack aggregates.
  • AI topic output is structured.
  • Caption follows SOP rules.
  • QA routes correctly.
  • Approved row appends to Content Calendar.
  • Revision row appends when rejected.
  • Lead-log task is created only after approval.
  • No auto-posting occurs.
